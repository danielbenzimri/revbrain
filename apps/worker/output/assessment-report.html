Audience: This report is designed for review of current-state CPQ complexity prior to planning.

1.1 Assessment Scope

This report provides a current-state assessment of the Salesforce CPQ environment. The following areas were assessed:

  • CPQ configuration objects (products, bundles, options, constraints, attributes)
  • Pricing rules, discount schedules, and custom scripts
  • CPQ package settings and plugin configuration

  • Advanced approval rules and workflows
  • 90-day transactional usage patterns and trends
  • User behavior and adoption metrics
  • Discount and override patterns
  • Data quality and field completeness
  • Technical debt indicators (dormant configs, duplicate rules)
  • Custom code: Apex, triggers, flows referencing CPQ objects
  • Complexity scoring across 5 weighted dimensions

5. Quote Lifecycle

The following lifecycle is derived from detected configuration patterns. Step sequence and frequency are inferred from metadata and have not been verified through direct process observation.

  1. Lead qualified → converted to Account, Contact, Opportunity.
  2. Sales Rep creates Quote from Opportunity.
  3. Quote Line Editor: products added, bundles configured with nested options. Product rules enforce selection and validation constraints.
  4. Pricing engine executes: price rules calculate adjustments, discount schedules apply tiered discounts, QCP custom JavaScript injects additional pricing logic.
  5. Approval routing via Advanced Approvals (sbaa) — multi-level chains with condition-based routing.
  6. Quote PDF generated from configured templates → DocuSign envelope created for e-signature.
  7. Quote accepted → Order auto-created.
